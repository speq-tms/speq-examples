# Contributing to speq-examples

## Workflow

- Add examples in a way that keeps both modes clear (`in-repo` and `test-repo`).
- Keep fixtures minimal and deterministic.
- Prefer realistic but non-sensitive API payloads.

## Commit style

Use Conventional Commit prefixes (`feat:`, `fix:`, `docs:`, `chore:`).

## Pull request checklist

- [ ] Example purpose is documented.
- [ ] Layout rules are respected.
- [ ] Related docs are updated.
- [ ] CI checks pass.
